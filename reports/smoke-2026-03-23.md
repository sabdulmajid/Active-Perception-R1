## Interpretation

- There is clear headroom from better perception policy: `oracle_crop` beats baseline on both models.
- Current prompt-only active loop underperforms baseline because these models rarely emit valid `zoom_roi` calls (`~0–4%` usage).
- This is a concrete, measured gap: the reward scaffold is present, but tool-use behavior is not yet aligned strongly enough to realize gains.

## End-to-End Engineering Status

- Live tool reinjection loop implemented in `src/active_perception_r1/sim/live_reinjection.py`.
- Benchmark runner now uses live reinjection instead of a single static two-pass shim.
- Unit tests pass: `11/11`.
- Remaining training blockers are now infrastructure/runtime-specific (model family support and optional backend deps), not missing core active-perception logic.
